# 深层提炼:理解文字背后的含义

> 2026-06-03 加。配图质量的瓶颈不在画技,在理解力。挑到了锚点但没吃透那个点背后的真意,画出来就是"表面图解"——正确但不传神。本档案的目标:从"图解段落"升级到"画出作者没直说的那层意思"。

## 为什么需要这一步

看两个例子(codex-platform 那批实际出图):

**传神的**:06-手术刀改表格(annotations)。文章讲的是"自然语言精准编辑数据"——表面画法是"小互在电脑上改数据",实际画了"小互拿着精密手术刀,趴在一张巨大的纸质表格上,用激光笔改一个格子,周围贴满'别动我'便签"。传神在哪?**抓住了"精准"这个张力**——不是大刀阔斧改,是在一堆不能动的东西中间,精确修一个点。"指哪改哪"四个字比讲三段技术原理都清楚。

**表面的**:07-三方赛跑(竞争格局)。文章讲 Anthropic/OpenAI/Microsoft 三家争企业市场。画了三条彩色赛道+三个卡通人跑步+终点旗。这就是"图解段落表面"——"三家在竞争"谁都知道,图没给出任何段落文字之外的理解。如果深挖一层:文章真正的判断是"Anthropic 先发但产品简单,OpenAI 后发但功能全,Microsoft 不自己做平台而是嵌入 Windows"——三家赌的东西完全不同,不是同一条赛道上的速度之争。这个"赌注不同"才是该画的。

## 三个强制问题(SKILL.md 1.3 步的展开)

### Q1:作者的真意是什么?

段落的文字是"说了什么",真意是"为什么在这里说这个"。

常见的真意模式:
- **判断**:"我认为 X 比 Y 重要" → 画 X 的重量/体量远超 Y
- **纠偏**:"大家都以为 A,其实是 B" → 画从 A 到 B 的认知翻转瞬间
- **预警**:"现在看着好,但..." → 画表面光鲜下面的裂缝/暗涌
- **类比**:"X 就像 Y" → 作者给了隐喻就用他的,但要画出类比中最精准的那一层而不是整个类比
- **总结提炼**:"说了一堆,核心就是这句" → 画这句话的视觉化,砍掉所有铺垫

**陷阱**:段落在列举细节(功能清单/数字/人名)时,真意通常不在细节本身,而在"这些细节合在一起说明了什么"。不要图解细节,画那个"说明了什么"。

### Q2:张力在哪?

没有张力的图是死图。张力 = 两种力在对抗/拉扯/反差。

找张力的方法:
- **翻转测试**:把段落的结论反过来说——"如果不是这样呢?"。反面越荒谬,正面的张力越强,图越有戏
- **期望差**:读者读到这里之前以为什么?读完之后认知变了什么?画这个"变"的瞬间
- **尺度反差**:小东西干了大事 / 大系统败给小细节 / 简单方法胜过复杂方案
- **时间反差**:曾经很重要现在不重要了 / 曾经不可能现在稀松平常
- **身份反差**:本来是工具现在变成队友 / 本来是创造者现在变成监工

张力找到后,图的构图自然就出来了:把对抗的两端都画进去,让读者一眼看出拉扯。

### Q3:读者看完这张图,脑子里应该留下的一句话是什么?

这句话的标准:
- **不是段落标题**(太泛)
- **不是段落摘要**(太长)
- **是一个能让人"啊"的顿悟**——如果这句话写在图上当唯一标注,读者看图+标注就能秒懂作者的核心判断,不需要回去读正文

好的例子:
- "指哪改哪"(annotations 图)→ 精准编辑的本质
- "不只写代码了"(coder-to-conductor 图)→ 身份转变的本质
- "参数真漂亮 / 但 CPU 没跑赢 M3 Max"(四格)→ 期望落差

差的例子:
- "三家公司在竞争"→ 这不是顿悟,这是事实陈述,谁都知道
- "Codex 有六种角色"→ 这不是顿悟,这是信息列举

**这句话就是这张图的灵魂**。后面编隐喻时,隐喻要服务这句话;选构图时,构图要突出这句话;写标注时,标注就是这句话或它的变体。

### Q4:这张图必须出现哪些来自原文的具体内容?(2026-06-10 加,治"传神但不准")

三问(真意/张力/灵魂话)解决的是"传不传神",但**传神 ≠ 准确**——一张让人"啊原来如此"的图,如果机制画歪了、关键部件缺了,比没图更糟,读者被带偏还以为懂了。所以三问之后,**回原文锁一张"必现内容清单":这张图非画不可的真实结构 / 部件 / 数字 / 步骤,逐条列出,每条 grep 一下原文确认(不靠记忆)。**

- **解释图**:列出它要呈现的真实部件,一个不能少。例 IFP 图:闪存、内存、共享专家、路由专家、按 prompt 路由——五个都得在,缺一个机制就不完整。
- **数字 / 术语**:必须是原文真值(45.6% 就写 45.6%,不能凑整;AFM 3 Cloud 不能写成 AFM Cloud),回 source 核对。
- **隐喻图**:列出隐喻每个物件**对应**机制的哪个真实部件(详见 metaphor-method 的"隐喻逐一对照")。

判据:图生成后,拿"必现清单"逐条核对——少一条 / 画错一条 = 不合格,重生。**灵魂话管"读者记住什么",必现清单管"读者别被带偏"——两个都要,缺了准确性,传神就是错得好看。**

## 深挖的实操技巧

1. **问"所以呢"**:段落说"Codex 现在有 500 万周活"——所以呢?所以它不再是程序员的玩具了,它是一个全民平台。画"全民平台"比画"500 万"有意思得多
2. **问"反过来呢"**:段落说"自然语言就能编辑数据"——反过来,以前是怎样的?以前要写代码、要懂 SQL。画"以前"和"现在"的反差,比画"现在能做什么"更有冲击
3. **问"谁最痛"**:段落讲一个技术变化——对谁影响最大?那个人的状态是什么?画那个人的状态,比画技术架构更打动人
4. **问"哪个细节最反直觉"**:一段话里总有一个细节是读者没想到的——抓住它放大,其他的缩小或不画。一张图只讲一个"没想到"

## 提炼结果怎么用

三个问题答完后,每个锚点应该有:
- 一个清晰的"真意"(不是段落摘要)
- 一个张力对(X vs Y)
- 一句灵魂话

带着这三样东西进 1.5(分轨)和 2(shot list)。shot list 里每张图的"核心意思"字段填的就是这个灵魂话,不是段落标题。编隐喻时,隐喻要把张力对可视化,不是把段落内容可视化。
