> 规矩:借鉴外部方法必须建档留血统——学底层原理,不照抄外观。这页记录本技能借了什么、没借什么、为什么。

## 借了什么(学的是方法/原理)

这套**工程化"主观创意流程"的四件套结构**,跟具体画风无关,可迁移:
1. **认知锚点筛选** → cognitive-anchors.md。不平均配图,只在"读者会停下来想"的点配图
2. **原创隐喻三步法**(抽象→物理动作→低科技物件→主体承担)+ 物件池/动作池 → metaphor-method.md。给"现编隐喻"上护栏
3. **反 PPT 负向防线 + 失败信号 + 迭代修法** → anti-ppt-qa.md。AI 默认滑向 PPT/素材库,用密集"不要什么"拽住
4. **reference 分层按需读** → SKILL 结构。主文件薄,重武器在 references
5. **反复刻规则**:案例只做风格校准不当模板,同类主题也换新隐喻——防止自己的案例库退化成模板

## 没借什么(那是人家的护城河,抄了就是抄人设)

- **小黑 IP 形象**:黑色实心小怪物。这是 Ian 的招牌,不碰
- **纯白手绘画风**:那是他的风格选择。本技能锁定的是**小互自己的专属画风(待定调)**,只继承"锁定一种风格换辨识度"这个**原理**,不继承外观

## 借鉴来源 2(2026-06-01):宝玉 baoyu-article-illustrator

- **项目**:JimLiu/baoyu-skills 的 baoyu-article-illustrator(无固定角色、纯样式库,与本技能并存)
- **GitHub**:https://github.com/JimLiu/baoyu-skills/tree/main/skills/baoyu-article-illustrator
- **看到时间**:2026-06-01(小互让我对比借鉴)

### 借了什么(单点做法,学原理)

1. **文字渲染铁律** → prompt-template.md「文字渲染铁律」+ anti-ppt-qa 复核项。两条经验规则:①色值/色名禁止被画成可见标签(模型会把 #hex 当文字画出来) ②图内文字错了不许代码涂改,只能改 prompt 重生/让用户挑候选
2. **每轨结构化 prompt 字段骨架**(ZONES/STEPS/NODES/LABELS/COLORS/STYLE/ASPECT,且 LABELS 必须填文章真实数字术语) → prompt-template 轨道二骨架。补强了本技能 1.5 图类型分流(原来只判轨道、没给图表轨配骨架)
3. **Density 配图密度显式可选**(精简/均衡/每节/丰富) → SKILL 第 2 步。给用户一个密度旋钮
4. **语义配色概念**(红=警示/绿=高效) → 已有情绪色,显式化进轨道二 COLORS 段

### 没借什么(冲突或过度设计)

- **EXTEND.md 持久化偏好**(记住 preferred_type/style/palette):与本技能"简单可重复、不记上次"的流程原则冲突,不抄
- **完整 Type×Style×Palette 三维 + preset 系统**:适合"无固定角色通用样式库";本技能 IP 锁定+皮肤固定+橙红主色,引入完整三维配色是过度设计。只借语义配色一条,不引入 Palette 维度
- **⚠️ 路线分叉(重要)**:宝玉用两条 ⛔ 明令"绝不用 SVG/HTML 代替位图、绝不在成图上代码叠字"。本技能**有意分叉**保留「文字层方案」——因他产出要跨 runtime 通用纯 bitmap,而小互终点是公众号、本就有排版层。但只限"标注零错字的解释图",且默认仍走纯位图重生(吸收了他的纯位图主张当默认,文字层当例外)

## 闭环

- 辨识度来自"不可复制的判断链 + 统一视觉",别人能抄方法,抄不了你的消化痕迹
- 本技能 = 小黑的方法骨架 × 小互的画风/场景/IP 决策
- 你 fork 本技能时,建议同样留一份自己的血统档:借了什么、没借什么、你的角色是怎么定的
