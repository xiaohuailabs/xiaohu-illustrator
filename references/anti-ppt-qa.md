# 怎么防止画成 PPT(负向防线 + 自检)

AI 配图默认就会滑向 PPT 和素材库风,必须用密集的"不要什么"把它拽住。本档案是生图后的守门人。

## ⛔ 适用范围(先分清,2026-06-10 加,治"反 PPT 把解释图也反没了")

反 PPT 不是一刀切,分两类图:
- **情绪锚点图 / 隐喻图**:严格反 PPT——别画成冷冰冰图表、别在左上写类型标题、要荒诞要现编隐喻。下面整套"绝对不要"主要管这一类。
- **解释图(流程/信息/对比/架构/时间线)**:目标是**准确 + 直观**,可以有清晰的 boxes / 箭头 / 标注 / 结构——这不算"画成 PPT",这是它的本分。对解释图,**下面这几条不适用**:"不要复杂架构图""不要把每个节点都解释清楚""太 PPT→改成手绘场景"。解释图该把机制的真实部件画全画对(见 deep-reading 必现内容清单 + metaphor 逐一对照),**清楚 > 怪诞**。
- **两类共同守**:反空洞凑图(没料别硬配)、图内不写色值色名/类型标题、中文标注零错字、全篇视觉契约统一。

一句话:反 PPT 反的是"为配而配的空洞图"和"情绪图画僵",**不是反"解释图把事讲清楚"**。

## 绝对不要(注入每条生图提示词的 constraints)

- 不要 PPT 信息图、不要正式流程图、不要课程课件
- 不要商业插画、不要精致扁平插画、不要科技感 UI、不要真实 App 截图
- 不要可爱卡通海报、不要儿童插画、不要表情包
- 不要复杂架构图、不要把每个节点都解释清楚
- 不要复杂背景、渐变、阴影、纸纹、噪点、米色复古纸感
- **不要在左上角写"Workflow / 系统架构图 / 常见坑 / 路线图"等类型标题**
- 不要在图上写出结构类型的名字

## QA 必过项

- 画幅符合这张按内容定的比例(横向内容才用 16:9,角色/纵向内容偏竖)且 prompt 方向词与之一致、干净白底(若 style-dna 定的是白底)
- 一张图只讲一个核心结构
- 主体不超过画面约 60%,留白充足
- 隐喻成立且是为这篇现编的(没复刻旧案例)
- 画面怪诞、有创意、有意思,不幼稚不卖萌
- 中文标注少、短、能读(最多 5-8 处,每处 2-8 字)
- 颜色克制(配色语义见 style-dna,定调后填)
- (IP 定调后)角色承担核心动作,不只是装饰
- **整组分布达标**(复核 SKILL 1.5 gate):情绪锚点图没超过一半;有结构内容的出了解释图;有过程/转折的出了四格漫画——若整组全是"小互+隐喻物件",说明没做图类型分流,打回重分
- **图内文字过关**(借鉴宝玉文字铁律):没有把色值/色名(#hex、"橙红")画成可见标签;关键标注(产品名/数字/步骤名)无错字。错了**改 prompt 减字重生**(新文件名留旧候选),⛔ 不许 PS/SVG/Pillow 涂改;标注必须零错字的解释图可走文字层方案(见 prompt-template)
- **全篇视觉契约统一**(风格一致复核,2026-06-01):所有图同一背景(白/浅底)、同一光影(柔和平光)、同一景深(清晰无虚化)、同一精致度(盲盒 icon)。任意两张并排看不出风格区别才算过;某张飘成电影感场景 / 景深虚化 → 重生(见 style-dna 视觉契约)

## 失败信号(命中就重生成或局部编辑)

- 左上角冒出类型标题
- 角色像吉祥物/表情包/可爱卡通
- 画面像 PPT、课件、正式流程图
- 元素太多、箭头太多、节点太多
- 文字变成大段解释
- 背景有纸纹/阴影/渐变/米色/噪点
- 中文错字严重或标注读不出
- 太死板,没有荒诞隐喻
- 跟旧案例构图太像

## 迭代修法(对症)

| 症状 | 修法 |
|---|---|
| 太普通 | 让主体成为动作核心,加一个奇怪但成立的隐喻 |
| 太复杂 | 删节点,只留一个动作 + 3-5 个短标注 |
| 表情不对内容 | 按 ip-character 映射表换表情,脸和动作要演出这张的情绪(焦虑/得意/心虚/笃定),不能一张呆脸 |
| 太 PPT | 去标题/边框/网格/过多箭头,改成手绘场景 |
| 太像旧案例 | 保留核心意思,换掉主物件和角色动作 |
| 文字错 | 优先局部编辑;错得多就重生成并减少标注数 |

## 交付判据(分两类,2026-06-10 改)

- **情绪 / 隐喻图**:读者先觉得"有点怪",然后 1 秒内看懂——第一眼像教程页 = 不合格。
- **解释图**:读者 1 秒看懂机制,且每个真实部件都对得上原文(回 deep-reading 必现清单逐条核对)。画得清楚利落不丢人,**"怪"不是目标,"准 + 直观"才是**。第一眼看不懂、或机制画错画漏 = 不合格。
